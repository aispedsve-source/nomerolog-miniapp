# Нумеролог — Telegram Mini App

Production-порт офлайн-приложения нумерологии (`org.glerus.nomerolog`) в веб / Telegram Mini App.
Все расчёты — в браузере, без сервера. Тексты трактовок **переписаны своими словами**
(оригинальные формулировки не копировались).

## Возможности (паритет с оригиналом)

Шесть разделов (нижняя навигация):

- **Карта** — полный персональный отчёт по дате рождения: сводка + 8 секций
  (Обзор, Сознание и точный день, Триггеры и тени, Кармическая задача, Деньги и реализация,
  Личный месяц/день, Возрастные циклы, Матрица даты). Кнопки «Поделиться» и «PDF».
- **Миссия** — миссия, сфера реализации, число сознания (детально).
- **Год** — личный год, месяц и день с рекомендациями.
- **Числа** — справочник 1–9: числа сознания, сферы реализации, личные годы.
- **Проф.** — профили (localStorage): добавить, открыть, удалить.
- **Экстра** (премиум) — совместимость пары (базовая + карта пары + глубокая),
  дата свадьбы, карта семьи, прикладные коды (имя/номер/дата события), PDF-отчёты.

Плюс: маска ввода даты `ДД.ММ.ГГГГ`, тема Telegram (тёмная/светлая), тактильный отклик,
шаринг, генерация PDF (тёмный фон, золото, A4) — полностью офлайн, без внешних библиотек.

### 🔮 ИИ-нумеролог (чат)
Плавающая кнопка на всех экранах открывает чат с ИИ-ассистентом: объясняет карту
**простыми словами**, задаёт вопросы, выявляет запрос клиента и мягко подводит к консультации
Мастера Васанты; помнит каждого клиента. Работает через отдельный бэкенд (Cloudflare Worker +
Mistral) — см. `../backend/`. Пока адрес бэкенда не указан в `js/config.js`, кнопка есть, но
чат сообщает, что подключается. Числа ИИ не считает — берёт готовую карту из движка.

## Структура

```
tg-miniapp/
├── index.html            # оболочка, 6 вкладок, Telegram SDK
├── css/style.css         # дизайн-система «Mystic» (точная палитра оригинала)
├── _devserver.py         # локальный dev-сервер с no-cache
└── js/
    ├── numerology.js     # движок расчётов (порт всех формул)
    ├── data.js           # словарь трактовок (оригинальные тексты, ~160 КБ)
    ├── extra-text.js     # тексты и логика пары/семьи/свадьбы (порт extra-калькуляторов)
    ├── report.js         # сборка персонального StructuredReport
    ├── render.js         # рендеринг StructuredReport → DOM
    ├── store.js          # профили + премиум (localStorage)
    ├── telegram.js       # обёртка Telegram WebApp SDK + Stars
    ├── pdf.js            # экспорт отчёта в PDF (canvas → PDF, без библиотек)
    ├── extra.js          # UI вкладки «Экстра»
    └── app.js            # навигация и оркестрация
```

Разбор всех формул — в `../ФОРМУЛЫ.md`. Извлечённый исходный словарь — `../dictionary_clean.json`.

## Запуск локально

```bash
cd tg-miniapp
python _devserver.py      # http://127.0.0.1:5178
```

(ES-модули не работают через `file://` — нужен http-сервер. `_devserver.py` отдаёт no-cache,
что удобно при разработке.)

## Деплой как Telegram Mini App

1. Выложить папку `tg-miniapp` на HTTPS-хостинг (GitHub Pages / Netlify / Vercel / Cloudflare Pages).
2. В [@BotFather](https://t.me/BotFather): `/newapp` → указать URL страницы.
3. Открывается кнопкой меню бота или по ссылке `t.me/<bot>/<app>`.

SDK уже подключён: тема, разворот, `HapticFeedback`, `BackButton`, шаринг. В обычном браузере
всё деградирует мягко.

## Монетизация (Telegram Stars)

Оригинал гейтил раздел «Экстра» одним булевым флагом (демо-доступ), с планом на RuStore Pay.
Здесь тот же флаг (`store.isPremiumUnlocked`), но разблокировка — через **Telegram Stars**:

- Кнопка «Открыть за ★» вызывает `tg.openInvoice(invoiceUrl)`.
- `invoiceUrl` в проде выдаёт **бот** через Bot API `createInvoiceLink` с валютой `XTR`.
  Пропишите его в `window.__NOMEROLOG_INVOICE_URL__` (или получайте с бэкенда бота).
- Для локальной проверки есть кнопка «Тестовый доступ (демо)».

Серверная проверка платежа (`successful_payment` у бота) должна подтверждать доступ —
клиентский флаг это лишь UX-слой.
